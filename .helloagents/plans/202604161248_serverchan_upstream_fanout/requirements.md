# XSZRemoteChatBridge Server酱3 消息接入 — 需求

确认后冻结，执行阶段不可修改。如需变更必须回到设计阶段重新确认。

## 核心目标
在现有游戏聊天上行链路和掉线提醒链路中新增 Server酱3 推送能力，使过滤命中的聊天消息与掉线提醒都可以在保持原机器人推送能力的同时，并行推送到 Server酱3；当机器人未配置或不可用时，Server酱3 仍可单独承担消息推送。

## 功能边界
- 复用现有聊天频道、关键词、全量频道和节流规则，命中后进入上行分发。
- 在现有机器人 `IngestEndpoint` 上行之外，新增 Server酱3 目标配置与发送流程。
- 机器人与 Server酱3 作为两个独立目标，允许单独启用、单独配置、单独告警。
- Server酱3 推送采用“简洁通知”样式：标题展示频道与发送者，详情包含世界信息与完整消息正文。
- 当机器人配置缺失但 Server酱3 已正确配置时，消息仍需正常推送到 Server酱3。
- 掉线提醒事件也需要走相同的目标分发能力，在 bot / Server酱3 任一目标可用时都能发送。
- 设置页允许用户配置并持久化 Server酱3 开关与发送地址，并清晰表达总开关与目标开关的关系。

## 非目标
- 不为 Server酱3 新增独立的频道过滤规则、关键词规则或独立匹配模式。
- 不改造下行链路（WebSocket / Pull）与机器人下行执行逻辑。
- 不引入新的外部服务依赖或复杂消息模板系统。

## 技术约束
- 维持现有 `C# / .NET 10 / Dalamud API 14` 技术栈。
- 兼容当前 `PluginConfiguration -> BridgeOptions -> SettingsWindow -> RemoteChatBridgeModule` 的配置与运行结构。
- Server酱3 发送不得阻塞游戏主线程，继续沿用现有异步网络发送模式。
- 不硬编码 SendKey 等敏感信息；配置仅从插件设置中读取，并避免在日志中输出完整凭据或完整推送 URL。
- 优先使用官方文档提供的 Server酱3 发送地址格式接入：`https://<uid>.push.ft07.com/send/<sendkey>.send`。
- 掉线提醒沿用现有 `EnableDisconnectReminder` 事件入口，不额外引入第二套事件检测逻辑。

## 质量要求
- `dotnet build plugin/XSZRemoteChatBridge.csproj -c Debug` 必须通过。
- 配置保存、重载后 Server酱3 设置必须保持一致。
- 对 bot / Server酱3 两个目标分别给出明确配置缺失告警，不再把“上行可用性”绑定到机器人配置。
- 任一目标发送失败不得阻断另一目标发送。
- 聊天上行与掉线提醒两条路径都必须遵守上述目标独立原则。
- 日志可定位目标状态，但不得泄露敏感配置。
