# XSZRemoteChatBridge Server酱3 消息接入 — 实施规划

## 目标与范围
本次实现聚焦两条现有推送路径的目标扩展：保留现有机器人推送能力，同时为过滤后的聊天消息和掉线提醒新增 Server酱3 推送目标。范围内需要完成配置模型、设置页、消息格式映射、目标分发、运行期告警与文档同步；范围外不处理下行链路和 Server酱3 独立过滤规则。

## 架构与实现策略
本次规划对比了三种方向：

- 方案 A：在现有 `UploadWithRetryAsync` 旁边直接追加一段 Server酱3 调用逻辑。
  不推荐。实现最短，但会继续把“上行”与“机器人配置”耦合在一起，无法优雅满足“机器人未配置但 Server酱3 仍可用”的约束。
- 方案 B：保留现有过滤与提醒事件，在发送前把消息分发到独立的目标发送器。
  推荐。只在“事件成立后、发送前”新增一层轻量分发判断，不重做现有桥接结构；bot / Server酱3 各自拥有独立配置校验、独立发送方法和独立告警，最符合当前需求。
- 方案 C：为 Server酱3 复制一整套独立过滤规则和配置界面。
  不推荐。灵活性更高，但明显超出本次需求边界，也会让设置页和运行逻辑复杂化。

推荐采用方案 B，具体落点如下：
- 将现有 `EnableUpstream` 调整为“上行消息总开关”的语义，保留其负责聊天监听与过滤的职责。
- 新增目标级开关，例如 `EnableBotUpstream` 与 `EnableServerChanUpstream`，默认保持 bot 目标开启以兼容当前行为。
- 将机器人发送保留在现有模块内，Server酱3 发送提取到新文件，例如 `plugin/ServerChanPushClient.cs`，避免继续扩张 `RemoteChatBridgeModule.cs` 的体积。
- 过滤通过后的聊天消息和掉线提醒都构造统一的发送上下文，分别判断 bot / Server酱3 是否“已启用且配置完整”，再独立发送；任一目标失败只记录自身日志，不影响另一目标。
- Server酱3 使用独立的消息格式化逻辑：标题为“频道 + 发送者”，正文使用多行详情承载世界与原始消息内容，优先采用 POST 发送，避免长文本拼 URL。
- 掉线提醒沿用同一 Server酱3 发送客户端，但使用单独的格式化方法，标题直接体现“掉线提醒”，详情包含角色和世界信息，避免误混入聊天通知样式。

## 完成定义
- 功能完成时，过滤命中的聊天消息可根据配置发送到 bot、Server酱3 或两者同时发送。
- 功能完成时，掉线提醒也可根据配置发送到 bot、Server酱3 或两者同时发送。
- 当 `EnableUpstream=true` 且仅 Server酱3 目标配置完整时，bot 缺失配置不会阻止 Server酱3 推送。
- 当 bot 目标配置缺失时，仅输出 bot 目标缺失告警；当 Server酱3 配置缺失时，仅输出 Server酱3 目标缺失告警。
- 设置页中用户可以清楚区分“上行消息总开关”和“目标开关”，并能填写 Server酱3 发送地址。
- Server酱3 推送内容符合已确认样式：标题体现频道与发送者，详情体现世界和完整正文。
- 掉线提醒的 Server酱3 推送内容需要与聊天消息区分，避免通知标题和正文歧义。
- 验证主路径：`review-first`
- reviewer 重点：
  - 上行分发是否真正解耦 bot 与 Server酱3 两个目标
  - 缺失机器人配置时 Server酱3 是否仍可工作
  - 掉线提醒是否复用同一目标分发能力，而不是继续绑定 bot 的旧判断
  - 日志是否避免泄露完整 Server酱3 推送 URL / SendKey
- tester 重点：
  - `dotnet build plugin/XSZRemoteChatBridge.csproj -c Debug`
  - 设置页保存并重载 Server酱3 配置
  - 聊天上行和掉线提醒两条路径在“仅 bot、仅 Server酱3、bot + Server酱3”组合下都按预期发送

## 文件结构
- `plugin/BridgeOptions.cs`
  新增 Server酱3 开关、发送地址和目标级开关；完善 Normalize 行为与默认值。
- `plugin/SettingsWindow.cs`
  调整“启用上行”文案为总开关；新增 bot / Server酱3 目标开关与 Server酱3 地址输入；保持自动保存行为。
- `plugin/RemoteChatBridgeModule.cs`
  将“过滤命中后直接上传 bot”和“掉线提醒直传 bot”重构为“事件产生后分发到可用目标”；独立校验目标配置并输出更精确告警。
- `plugin/ServerChanPushClient.cs`（新）
  负责构造 Server酱3 请求、聊天消息/掉线提醒两类正文格式化、发送与错误处理。
- `README.md`
  补充 Server酱3 上行配置说明。
- `helloagents/CHANGELOG.md`
  记录 bot / Server酱3 双目标上行能力。
- `helloagents/wiki/modules/plugin.md`
  更新插件模块文档中的上行目标与配置语义。

## UI / 设计约束
本次不是视觉重做任务，维持现有 ImGui 设置页风格。重点是信息结构清晰：
- “启用上行消息推送”作为总开关放在原基础开关区。
- bot 与 Server酱3 目标配置归于同一语义区域，避免用户误以为 `EnableUpstream` 仍然只控制机器人。
- Server酱3 地址字段文案应明确提示使用官方 send URL，减少用户把普通文档页 URL 误填为接口地址。

## 风险与验证
- 风险：bot 和 Server酱3 发送逻辑仍然耦合，导致任一目标异常时影响另一个目标。
  验证：代码审查中确认目标配置判断与异常处理边界；手工验证三种目标组合。
- 风险：掉线提醒仍保留旧的 bot 专用判断，导致与聊天消息的目标行为不一致。
  验证：审查掉线提醒入口是否改为复用统一目标分发方法。
- 风险：Server酱3 URL 或 SendKey 被直接写入日志。
  验证：审查新增日志语句，仅记录“已配置/未配置”或脱敏后的目标信息。
- 风险：Server酱3 正文过长或格式不稳定。
  验证：使用 POST 发送并集中在格式化方法里构造标题/详情，避免散落拼接。
- 风险：旧用户升级后 bot 行为被意外关闭。
  验证：新目标开关默认值与现有上行行为保持兼容，构建后进行配置迁移检查。
