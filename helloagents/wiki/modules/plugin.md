# plugin

## 目的
提供 FF14 与机器人 / Server酱 之间的消息推送能力，以及机器人下行回游戏的桥接能力。

## 模块概述
- **职责:** 监听游戏聊天、执行关键词与频道过滤、向机器人 / Server酱 发送推送、执行下行消息与配置管理。
- **状态:** ✅稳定
- **最后更新:** 2026-03-11

## 规范

### 需求: 自动化发布与分发
**模块:** plugin
保证插件产物可被持续构建并可被 Dalamud 插件源消费。

#### 场景: 主分支推送后自动发布
触发条件：主分支出现新提交。
- 自动完成编译并生成可分发压缩包。
- 分发仓库中的 `pluginmaster.json` 与插件包地址保持同步。

### 需求: 关键词按频道独立过滤
**模块:** plugin
支持每个关键词独立勾选可上行的聊天频道，并在“任意命中”模式下默认使用正则匹配。

#### 场景: 不同关键词对应不同频道
触发条件：收到上行聊天消息。
- 若存在 `KeywordChannelRules`，按“当前频道可用关键词集合”执行匹配。
- 若不存在映射规则，则回退到“所有频道 + `KeywordRules`”兼容逻辑（不再依赖全局频道白名单）。
- 若未配置任何关键词规则且未命中全量频道，则该消息不上报。

#### 场景: 任意命中默认正则
触发条件：关键词匹配模式被设置为“任意命中”。
- 自动启用 `KeywordUseRegex`。
- UI 中该选项显示为启用且不可关闭，避免规则语义与运行时不一致。

#### 场景: 关键词管理界面操作
触发条件：用户在设置页维护关键词映射。
- 左侧显示关键词列表，支持 `+/-` 快速增删并切换当前编辑项。
- 右侧显示当前关键词详情，支持编辑关键词文本与频道 ID 过滤勾选。

#### 场景: 自动保存静默提示
触发条件：设置页自动保存并重载配置。
- 不向游戏聊天频道输出“配置已保存并重新加载”提示，避免刷屏。
- 仅保留插件内部日志记录（Information）。

### 需求: 指定频道全量上行
**模块:** plugin
支持将指定聊天频道的全部消息直接上报机器人，不受关键词规则限制。

#### 场景: 全量上行频道命中
触发条件：收到上行聊天消息且消息频道命中 `UploadAllChannelList`。
- 跳过关键词规则解析与匹配，直接进入上行发送流程。
- 仍保留节流、签名与重试机制，确保与普通上行链路一致。

### 需求: 双目标消息推送
**模块:** plugin
支持将聊天上行与掉线提醒发送到机器人和 / 或 Server酱，且两个目标独立配置、独立告警、互不阻塞。

#### 场景: 聊天消息推送到独立目标
触发条件：收到命中过滤规则的上行聊天消息。
- 当 `EnableBotPush=true` 且机器人配置完整时，消息推送到机器人。
- 当 `EnableServerChanPush=true` 且 `ServerChanSendUrl` 配置完整时，消息推送到 Server酱。
- 任一目标失败不得阻断另一目标。

#### 场景: 仅 Server酱 可用
触发条件：未启用机器人目标，或机器人配置缺失，但 Server酱 目标已启用且配置完整。
- 聊天上行仍可发送到 Server酱。
- 掉线提醒仍可发送到 Server酱。

#### 场景: 掉线提醒复用目标分发层
触发条件：检测到连接中断弹窗且 `EnableDisconnectReminder=true`。
- 掉线提醒沿用与聊天上行一致的目标可用性判断。
- Server酱 使用独立的掉线提醒标题与详情格式，避免与聊天消息混淆。

#### 场景: 启动时快速识别推送目标缺失
触发条件：插件初始化桥接模块。
- 聊天上行或掉线提醒未启用任何推送目标时输出明确告警。
- 机器人目标缺少 `IngestEndpoint/BridgeKey/BridgeSecret` 时输出机器人目标告警。
- Server酱 目标缺少 `ServerChanSendUrl` 时输出 Server酱 目标告警。

#### 场景: 设置页查看 Server酱 使用说明
触发条件：用户打开设置页中的 `Server酱` 页签。
- 页面展示简明接入步骤与 Send URL 示例。
- 提供官方首页、SendKey 页面、使用说明、APP 安装和 API 文档链接。
- 点击链接后直接在系统默认浏览器打开对应网址。

#### 场景: 设置页配置全量频道
触发条件：用户在“上行过滤”中维护全量上行频道列表。
- 可按频道中文名/枚举名/频道 ID 搜索并勾选。
- 配置自动保存并立即生效。

#### 场景: 全量频道文案统一
触发条件：用户在设置页查看上行过滤配置。
- 将配置项文案统一为“指定频道消息全部上传（无视关键词）”。
- 删除“全局频道白名单”入口，避免与关键词映射能力重复。

### 需求: WS 下行会话稳定性与可观测性
**模块:** plugin
降低下行链路在网络抖动或反向代理空闲超时场景下的断连频率，并提升断连诊断效率。

#### 场景: WS 空闲连接保活
触发条件：WS 会话建立并鉴权成功。
- 按 `WsPingIntervalSeconds` 主动发送应用层 `ping`。
- 若超过 `WsPongTimeoutSeconds` 未收到任何服务端帧则主动中断会话并触发重连。

#### 场景: WS 被服务端关闭时定位原因
触发条件：收到 WebSocket Close 帧。
- 告警日志包含 `CloseStatus` 与 `CloseStatusDescription`。
- 重连前保留一次 pull 回退能力，降低下行消息丢失概率。

#### 场景: 同 key 会话冲突
触发条件：WS 关闭原因为 `replaced` 或 `already_connected`。
- 日志提示“同 bridge_key 连接冲突”。
- 提高重连退避等级，避免冲突状态下的快速重连刷屏。
- 对同类冲突告警进行节流，减少重复 Warning 日志噪声。

#### 场景: 启动时快速识别配置缺失
触发条件：插件初始化桥接模块。
- 上行缺少 `IngestEndpoint/BridgeKey/BridgeSecret` 时输出明确告警。
- 下行缺少 `WebSocketEndpoint/BridgeKey/BridgeSecret/PullEndpoint` 时输出明确告警。

#### 场景: 非命令下行消息发到游戏频道
触发条件：机器人下发内容不以 `/` 开头，且未生成命令前缀。
- 采用 `UIModule.ProcessChatBoxEntry` 注入聊天输入流程（与 OmenTools 同源实现）。
- 下行日志记录最终执行文本，便于定位执行异常。

#### 场景: 下行执行结果判定
触发条件：插件调用命令系统执行下行指令。
- 斜杠命令通过 `ShellCommandModule.ExecuteCommandInner` 执行。
- 普通文本通过 `UIModule.ProcessChatBoxEntry` 执行。
- 任一路径抛错时记录“下行执行失败”并附带最终文本。

## API 接口
- `POST /ff14/bridge/ingest`
- `POST /ff14/bridge/pull`
- `WS /ff14/bridge/ws`
